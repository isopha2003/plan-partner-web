## 0. 문서 목적

Figma 디자이너가 화면 설계를 시작할 수 있도록:
- 무엇을 만드는 서비스인지

## 1. 제품 개요

**한 줄 정의**: 하루/주/달 단위로 시간대에 "블록"을 배치해 계획을 세우고, 실제 집중 시간을 타이머로 측정하며, 완료 기록을 월간 캘린더 형태로 쌓아가는 생활 관리 웹 서비스.

**핵심 가치**:
- 계획(오늘 배치된 블록들의 총 계획 시간)과 실제(전역 타이머로 측정한 하루 총 집중 시간)를 한 화면에서 비교
- 반복되는 활동은 "블록 템플릿"으로 미리 만들어두고 드래그로 재사용 → 매번 새로 만들 필요 없음
- 활동 기록을 월간 캘린더에 날짜별로 쌓아, 클릭 없이도 그날 무엇을 했고 얼마나 집중했는지 바로 확인 🔄 v2(원래는 깃허브 스타일 잔디 그리드였음, 아래 5.9 참고)

**이번 단계 범위**: 웹 우선 출시. 데스크톱 브라우저를 기준으로 설계하되, 이후 반응형으로 태블릿/모바일 웹까지 커버. 네이티브 앱은 별도 프로젝트로 추후 진행(참고: `D:\plan_partner_project`에 Flutter 앱 스펙 존재, 이번 웹 프로젝트와는 독립적으로 취급).

| 화면 꺼짐/켜짐 기반 타이머 자동 정지·재시작 (3.2) | 🔄 웹에서는 "화면 on/off" 대신 **브라우저 탭 활성 상태(Page Visibility API 기준)**로 재정의 제안: 이 탭을 벗어나 다른 탭/창으로 전환하면 화면 꺼짐과 동일하게 취급 → 자동 일시정지 상태였다면 탭이 다시 활성화될 때 자동 재시작. 수동 정지 상태는 기존과 동일하게 탭 전환과 무관하게 아무 반응 없음 |

| 뽀모도로/마감 알림 | 🔄 동일하게 브라우저 알림으로 대체 |
| 가로모드 상시 디스플레이 (3.4) | ❌ 원본에서도 보류 상태였고, 모바일 가로모드 자체가 웹에 해당 없음. 웹에서는 대신 "타이머를 작게 띄워 다른 작업하면서 볼 수 있는 미니 위젯(플로팅 타이머)"을 🆕 대체 아이디어로 제안 — 확정 아님, 필요시 별도 논의 |
| 템플릿 팔레트의 "확장형 바텀시트" (1번) | 🔄 모바일 바텀시트는 데스크톱에 맞지 않는 패턴. 웹에서는 **화면 하단 또는 좌측에 고정된 템플릿 패널**로 대체 제안: 평소엔 가로 스크롤 한 줄(또는 접힌 좁은 패널)로 보이다가, 클릭 또는 위로 드래그 시 패널이 확장되어 그리드로 더 많은 템플릿을 보여줌. 모바일 웹 화면에서는 원본처럼 바텀시트 유지 가능 |
| 터치 드래그 앤 드롭 | 🔄 마우스 기반 드래그 앤 드롭으로 구현(데스크톱), 터치 디바이스에서는 터치 드래그도 지원 |

## 5. 핵심 화면 명세

### 5.1 오늘 화면 (기본 진입 화면) ✅

**목적**: 로그인/접속 시 가장 먼저 보이는 화면. "오늘 뭘 해야 하고 뭘 했는지"를 한눈에.

**구성 요소**:
- 오늘 날짜의 블록을 **시간순으로 나열한 리스트**
- 각 항목 옆에 **완료 체크박스** (클릭 즉시 완료 처리, 이때 활동 기록 캘린더(5.9)에도 반영됨을 내부적으로 인지하되 화면상 별도 확인 절차 없음)
- 오늘 마감인 마감 작업 + **지난 마감(미완료)** 작업도 같은 리스트에 함께 표시 (마감 작업은 시각적으로 구분되어야 함 — 예: 아이콘 또는 태그로 "마감" 표시, 지난 마감은 강조색)
- 화면 상단(또는 항상 보이는 위치)에 **전역 타이머 위젯** 배치 — 특정 블록에 종속되지 않는, 하루 전체 집중 시간을 측정하는 단일 타이머 (5.5 참고)

**상태**:
- 오늘 블록이 없는 빈 상태 (예: "오늘 계획된 활동이 없어요 + 캘린더로 이동 버튼")
- 전부 완료한 상태(축하 메시지 등 고려 가능, 확정 아님)

### 5.2 캘린더 화면 ✅

**목적**: 블록을 배치하고 관리하는 메인 작업 공간.

**구성 요소**:
- 기간 선택: **일 / 주 / 달 / 년** 전환 컨트롤
- **그리드 뷰 ↔ 리스트 뷰 토글**
  - 그리드 뷰: 시간대에 비례한 블록 배치(세로축 시간, 블록 높이 = 길이). 2시간 블록은 1시간 블록보다 시각적으로 2배 크게
  - 리스트 뷰: 선택한 날짜의 블록을 시간 표시 없이 단순 체크리스트로 표시 (5.1 "오늘" 화면과 동일한 리스트/완료 UI를 재사용, 임의의 날짜에 적용)
- **템플릿 패널** (섹션 3 참고): 블록 템플릿 목록, 드래그 시작점. 평소 축소 상태 / 확장 시 그리드로 더 많은 템플릿 노출
- 시간 겹치는 블록끼리는 **경고 표시** (예: 테두리 색 강조, 경고 아이콘)
- 마감 작업은 캘린더 뷰에 **날짜 마커**로 표시 (시간대 없는 항목이므로 하루 전체에 걸친 마커/배지 형태)

**핵심 인터랙션** (디자이너가 반드시 시각화해야 할 플로우):
1. 템플릿 패널에서 템플릿을 캘린더의 특정 시간대로 **드래그 → 드롭**하면, 별도 시간 입력 화면 없이 **즉시 인스턴스 생성** (기본 길이로 배치, 15분 단위 스냅)
2. 생성된 블록 인스턴스의 **위/아래 가장자리를 드래그**하면 시작/종료 시간이 조정됨(리사이즈). 시간 입력은 오직 이 방식으로만 — 별도 시간 피커 모달 없음
3. 블록 인스턴스를 클릭하면 상세 화면/패널(5.4) 오픈 — ✅(2026-07-02 결정) **사이드 패널** 방식. 화면 한쪽(우측 또는 좌측)에 패널이 열리고 캘린더 그리드는 그대로 유지되어, 캘린더 맥락(앞뒤 일정)을 보면서 상세를 편집할 수 있음. 모달 아님
4. 습관 스태킹으로 연결된 블록들은 캘린더 그리드 위에 **연결선(선/화살표)으로 직접 시각화** — ✅(2026-07-02 결정, 5.10 참고)

### 5.3 블록 템플릿 생성/편집

**목적**: 재사용 가능한 활동 정의(제목 + 색상 + 태그)를 만들고 관리.

**구성 요소**:
- 제목 입력
- 색상 선택 (색상 변경 시 해당 템플릿을 쓰는 **모든 기존 인스턴스에도 반영**됨을 UI로 암시할 필요는 없으나, 색상 피커는 명확하게)
- 태그 선택/생성 (템플릿 단위로 부여 — 인스턴스가 아님)
- 저장/삭제

### 5.4 블록 인스턴스 상세 (자식 블록 · 메모)

**목적**: 캘린더에 배치된 개별 블록을 열었을 때 보이는 상세 화면. ✅ **사이드 패널**로 구현(모달 아님) — 캘린더 그리드는 유지된 채 화면 한쪽에 패널이 열려서, 앞뒤 일정을 보면서 편집 가능

> 🔄 v2 변경: 블록별 개별 타이머는 없앰(아래 참고). 이 화면에는 더 이상 타이머가 없고, 시간 관련 정보는 계획된 시작~종료 시간뿐임.

**구성 요소**:
- 상단: 제목(템플릿에서 상속), 색상, 날짜, 시작~종료 시간
- **자식 블록 목록**: 아래 두 형태를 모두 지원해야 함
  - 체크리스트형 자식: 텍스트 + 체크박스만, **무제한 중첩** 가능
  - 독립 타임블록형 자식: 부모처럼 자체 시작/종료 시간을 가지며(캘린더 상 별도 시간대 배치용) 템플릿 참조. **부모→자식 1단계까지만** (자식의 자식으로 타임블록을 또 넣을 수는 없음 — UI에서 "자식 블록 추가" 시 타임블록 옵션은 최상위 블록에만 노출되고, 이미 자식인 타임블록 안에서는 체크리스트만 추가 가능하도록 제약)
- **메모 영역**: 자유 텍스트 (예: "7장 페이징, 8장 가상메모리 정리함")
- **습관 스태킹 연결**: "이 블록 끝나면 다음 블록" 체인 설정 UI (다음 블록 선택)
- 완료 체크박스

### 5.5 전역 타이머 위젯 (3-state) 🔄 v2

**목적**: 특정 블록에 종속되지 않는 **단일 전역 타이머**. "이 블록을 얼마나 했는지"가 아니라 "오늘 총 얼마나 집중했는지"만 측정. 블록마다 시작/정지 버튼을 두지 않고, 어느 화면에서든 접근 가능한 상시 노출 위젯(예: 오늘 화면 상단, 또는 전 화면 공통 헤더) 하나로 통합.

> 🔄 v2 변경 이유: 활동별 시간 구분을 포기하고 "총 집중 시간"만 측정하는 방향으로 결정됨(원래는 블록별 개별 타이머였음). 이에 따라 목표 대비 실제 비율(5.9)도 활동별이 아닌 하루 단위 총합 비교로 변경됨.

**디자인에서 반드시 3가지 상태를 시각적으로 구분**해야 함 (이 로직 자체는 원안 유지, 적용 대상만 블록→전역으로 변경):

| 상태 | 진입 조건 | 웹에서의 화면 밖 동작 |
|---|---|---|
| 실행중 | 사용자가 시작 버튼 클릭 | 탭이 다시 활성화되면 → 자동 일시정지로 전환 |
| 자동 일시정지 | 탭이 비활성화(다른 탭/창으로 전환)되어 시스템이 자동으로 멈춤 | 탭이 다시 활성화되면 → 자동으로 실행중 복귀 |
| 수동 정지 | 사용자가 직접 정지 버튼 클릭 | 탭 전환과 **무관하게 아무 반응 없음**, 재시작은 사용자가 직접 |

- 세 상태는 색상/아이콘으로 명확히 구분되어야 함 (예: 실행중=초록 pulsing, 자동정지=노랑, 수동정지=회색)
- 방치 알림: 수동 정지가 일정 시간(사용자 설정 가능) 지속되면 브라우저 알림 발송 → 알림 설정 UI 필요 (몇 분 후 알림 보낼지)
- 자동재시작 시 확인 팝업: ❌ 없음 (조용히 자동 전환)
- 🆕 브라우저 탭 타이틀에 실시간 타이머 표시 (섹션 3 참고, 디자인 영향은 적지만 개발 시 참고용으로 명시)

### 5.6 뽀모도로 모드 설정 ✅

- 온/오프 토글
- 공부시간 / 쉬는시간 직접 입력 (고정값 아님)
- 설정 시간 도달 시 알림 발송

### 5.7 마감 작업 목록 ✅

**목적**: 시간대 없이 날짜(마감일)만 있는 작업 관리.

**구성 요소**:
- 작업 목록 (제목 + 마감일)
- 상태 2가지: **진행중** / **마감 지남(미완료, 강조 표시 — 예: 빨간색)**
- "아직 끝내지 못한 일"(5.8)과는 시각적으로도 완전히 구분되는 별도 목록 — 혼동되지 않도록 레이블링에 주의
- 완료 체크 또는 삭제 시 목록에서 제거
- 시간대 지정 버튼 → 클릭 시 일반 타임블록으로 전환(캘린더에 배치하는 플로우로 연결)

### 5.8 "아직 끝내지 못한 일" ✅

**목적**: 과거에 시간대를 정해 계획했지만 완료 못 한 블록 모음 ("계획 vs 실제" 실패 기록).

- 목록형 UI, 완료 체크 또는 삭제 시 사라짐
- 5.7 마감 작업과 개념이 다름을 레이블/설명 문구로 명확히 구분

### 5.9 활동 기록(캘린더형) & 통계 화면 🔄 v3

> 🔄 v3 변경(2026-07-02): 원안(깃허브 스타일 잔디 그리드)을 **월간 캘린더 형식**으로 교체 — 잔디는 클릭해야 그날 내용을 볼 수 있는데, 그보다 날짜별 활동/시간이 클릭 없이 바로 보이는 게 낫다는 판단. 트레이드오프: 잔디 그리드가 가진 "1년 전체를 한 화면에서 스캔"하는 장점은 약해짐 — 이를 보완하기 위해 목표 달성일에는 셀에 가벼운 강조 표시를 남겨두는 것을 제안(아래).

**구성 요소**:
- **월간 캘린더 그리드** (요일 x 주 배열의 일반적인 달력 형태, 5.2 캘린더 화면과 톤은 다르지만 레이아웃 패턴은 재사용 가능)
- **각 날짜 셀에 인라인으로 표시**: 그날 완료한 활동명 목록(간단히, 예: "공부", "운동") + **총 집중 시간**(예: "3h 20m"). 클릭 없이 바로 읽을 수 있어야 함
  - 완료 활동이 많아 셀에 다 못 들어가는 날은 일부만 보여주고 "+N개" 표시 → 클릭 시 전체 목록을 보여주는 팝오버(잔디의 "클릭해서 상세보기"는 이 경우에만 보조적으로 남음)
- 기준은 계획된 날짜가 아니라 **실제 완료 처리한 날짜** (기존 원칙 유지)
- 🆕 목표 달성한 날(하루 총 집중 시간이 목표 이상)은 셀에 가벼운 강조 표시(배경색 틴트 또는 작은 배지) — 잔디의 "한눈에 스캔" 장점을 일부 보완하는 용도, 확정 아님(디자이너 검토 필요)
- 월 이동은 이전/다음 달 화살표로 (9번 오픈 퀘스천 참고)
- **오늘 기준 체크리스트 달성률(%)** 시각화 (예: 원형 게이지 or 프로그레스 바)
- **시간 목표 대비 실제 집중 시간 비율(%)** 시각화 — **하루 단위**로 비교. 목표 시간은 **하이브리드 방식** ✅(2026-07-02 결정): 그날 캘린더에 배치된 블록들의 총 계획 시간을 기본값으로 자동 제안하되, 사용자가 숫자를 직접 수정해서 덮어쓸 수 있음(예: 자동 제안 4시간 → 사용자가 6시간으로 수정). 수정값은 그날에만 적용(매일 새로 자동 제안값이 뜨고, 필요시 다시 수정). 실제 시간은 전역 타이머로 측정한 하루 총 집중 시간
- 스트릭 프리즈, 태그별 레벨 시스템: ❌ 제외

### 5.10 습관 스태킹(루틴 체인) 시각화

✅(2026-07-02 결정) 별도 화면 없이 **캘린더 그리드 위에 직접** 시각화. 체인으로 연결된 블록들(보통 시간상 연속 배치된 경우가 많음 — 예: 기상→운동→샤워)을 선/화살표로 이어서 표시. 연결 편집은 블록 상세 패널(5.4)의 "다음 블록 선택" UI에서 수행.

- 유의점: 캘린더에 블록이 많은 날에는 연결선이 다른 블록들과 겹치거나 지저분해질 수 있음 — 디자이너가 선 스타일(곡선/직선), z-index, 호버 시에만 강조 등으로 가독성 확보 필요

### 5.11 설정 화면

- 방치 알림 임계 시간
- 뽀모도로 기본값
- 브라우저 알림 권한 안내/설정 유도

## 10. 버전 변경 이력

- **2026-07-02**: 타이머를 블록별 개별 타이머에서 **전역 단일 타이머**로 변경. 활동별 시간 구분(어떤 활동에 얼마나 썼는지)은 포기하고, 하루 총 집중 시간만 측정하는 방향으로 결정. 영향 범위: 5.4(블록 상세에서 타이머 제거), 5.5(전역 위젯으로 재정의), 5.9(목표 비율을 하루 단위 총합 기준으로 변경). 자식 블록의 "독립 타임블록형"은 시간대 배치(스케줄링) 목적은 유지하되 개별 타이머 보유는 아님.
- **2026-07-02**: 하루 목표 집중 시간 산출 방식을 **하이브리드**로 결정 — 그날 배치된 블록들의 총 계획 시간을 기본값으로 자동 제안하고, 사용자가 숫자를 직접 수정 가능(그날 한정 적용). 섹션 5.9, 6번 컴포넌트 목록 반영.
- **2026-07-02**: 블록 인스턴스 상세는 **사이드 패널**로, 습관 스태킹 체인은 **캘린더 위 연결선**으로 직접 시각화하기로 결정(별도 화면 없음). 섹션 5.2, 5.4, 5.10 반영. 오픈 퀘스천에서 두 항목 제거.
- **2026-07-02**: 5.9의 **깃허브 스타일 잔디 그리드를 월간 캘린더 형식으로 교체** — 날짜별 완료 활동 목록 + 총 집중 시간을 클릭 없이 셀에 바로 표시. 목표 달성일 강조 표시를 잔디의 "한눈에 스캔" 장점을 보완하는 용도로 제안(🆕, 확정 아님). 섹션 1, 4, 5.1, 5.9, 6, 8 반영.

## 11. 용어 정리 (원본 기획과 동일)

- **블록 템플릿**: 제목+색상을 가진 재사용 가능한 활동 정의 (예: "공부")
- **블록 인스턴스**: 템플릿을 캘린더 특정 시간대에 배치해서 생긴 실제 항목 (날짜·시간·완료여부 보유)
- **템플릿**(8번 관련, 위와 다른 개념): 하루/주 전체 일정을 복제하는 기능
- **마감 작업**: 시간대 없이 날짜만 지정된 작업
- **아직 끝내지 못한 일**: 과거에 시간대를 정해 계획했지만 미완료로 남은 것
